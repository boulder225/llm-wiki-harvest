Create `src/claude_dump/fireflies_client.py` with class `FirefliesClient`, mirroring `ClaudeAPIClient` structure:

Constructor `__init__(self, api_key: str, verbose: bool = False)`:
- Create `self._http = httpx.Client(base_url="https://api.fireflies.ai", timeout=httpx.Timeout(30.0, connect=10.0), headers={"Authorization": f"Bearer {api_key}", "Content-Type": "application/json"}, follow_redirects=True)`
- Store `_verbose` and `_console = None` (lazy Rich console)

Context manager: `__enter__`, `__exit__`, `close()` — identical pattern to ClaudeAPIClient.

Private `_graphql(self, query: str, variables: dict | None = None) -> dict`:
- POST to `/graphql` with JSON body `{"query": query, "variables": variables or {}}`
- Retry logic: same constants (`_MAX_RETRIES = 5`, `_INITIAL_BACKOFF_SECONDS = 2.0`), same retryable codes (`{429, 500, 502, 503}`)
- On 401/403: raise `FirefliesAuthError()` immediately (no retry)
- On success: parse JSON, check for `"errors"` key in GraphQL response — if present and `data` is None, raise `FirefliesAPIError(status_code=200, response_body=str(errors))`
- Return the `"data"` dict from the response
- Include `_backoff_delay`, `_parse_retry_after`, `_log_retry` — copy from ClaudeAPIClient

Public methods:

`list_transcripts(self, limit: int = 50, skip: int = 0) -> list[FirefliesTranscriptSummaryItem]`:
- GraphQL query: `query Transcripts($limit: Int, $skip: Int) { transcripts(limit: $limit, skip: $skip) { id title date duration transcript_url participants } }`
- Variables: `{"limit": limit, "skip": skip}`
- Parse `data["transcripts"]` with `FirefliesTranscriptSummaryItem.model_validate`

`list_all_transcripts(self) -> list[FirefliesTranscriptSummaryItem]`:
- Paginate: call `list_transcripts(limit=50, skip=offset)` in a loop, incrementing skip by 50 each time, until fewer than 50 returned
- Returns complete list

`get_transcript(self, transcript_id: str) -> FirefliesTranscript`:
- GraphQL query: `query Transcript($transcriptId: String!) { transcript(id: $transcriptId) { title date duration transcript_url audio_url video_url speakers { id name } meeting_attendees { displayName email } sentences { speaker_name text raw_text start_time end_time } summary { keywords action_items overview shorthand_bullet outline } } }`
- Variables: `{"transcriptId": transcript_id}`
- Parse `data["transcript"]` with `FirefliesTranscript.model_validate`
- Set `result.id = transcript_id` after parsing (the detail query doesn't return `id` in the response)

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-05-02 | T (Tampering) | fireflies_models.py | mitigate | Pydantic model_validate with extra="ignore" rejects unexpected shapes; no dynamic code execution from API data |
