<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> Claude.ai API | File download responses are untrusted binary data |
| API response -> filesystem | File names from API written to local paths |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | File download path | mitigate | Filename uses `{uuid[:8]}_{original_name}` -- the uuid prefix is safe (hex chars). The original_name comes from the API and could contain path traversal characters. Sanitize by stripping any `/`, `\`, `..` sequences from `ref.file_name` before building the path. Use `Path.name` to extract just the filename component. |
| T-03-02 | Denial of Service | File download size | accept | Files come from the user's own Claude.ai project. No external attacker can inject large files. Accept risk. |
| T-03-03 | Information Disclosure | Knowledge doc content | accept | Knowledge docs are the user's own files. Writing them locally is the intended behavior. |
| T-03-04 | Tampering | Knowledge doc file_name | mitigate | Same path traversal risk as T-03-01. Use `Path(doc.file_name).name` to strip directory components from knowledge doc filenames before writing. |
</threat_model>
